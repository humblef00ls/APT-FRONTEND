APT: Moving

Specification

filename: Moving.py def boxes(items, weights, n): ''' items (list of str) - List of item names weights (list of int) - List where the kth element is the weight of the kth item in the list items n (int) - Number of boxes to pack the items into Return a list of strings where each string represents the items to pack in that box. ''' return []

Problem Statement

To make packing faster you've decided to implement the function boxes that tells you what item goes in which box and the order to put the items in (heaviest first). You have a list of items and a parallel list of weights, where the kth item in items has the kth weight in weights. You must use n boxes to pack things.

When choosing which of n boxes to put an item in, you always choose the lightest/least heavy box, even if it has more items in it. You put items into each box in order of individual items from heaviest to lightest. First put the heaviest item in the lightest box, then the next heaviest item in the lightest box, and so on. See below for tie-breaking information.

The function returns a list of strings, where each string represents a single box of all the items that go in it, separated by a single space. In each single box (a string) the space-separated items are listed from heavest to lightest. The list of strings returned is in alphabetical order.

To create your return value, first sort the items by weight with heaviest first, break ties alphabetically. For example, a book with weight 2, a doll with weight 1, and a car with weight 1 would be ordered book, car, and doll because the book is the heaviest and although car and doll weigh the same, car comes before doll alphabetically. Process these weighted items in order, always putting the next heaviest item in the box that is the lightest. If there is more than one box that is the lightest, use the first one in your list you're returning.

Constraints

  • len(items) == len(weights)

  • n <= len(items)

  • There will be no more than 50 elements in the list parameters.

Examples

  1. items = ['pens', 'mug', 'book']
weights = [1, 3, 2]
n = 3

Returns ['book', 'mug', 'pens']
    # When we order the items by weight we get ['mug', 'book', 'pens'], then we put them in our boxes to get the list ['mug', 'book', 'pens'], finally we return ['book', 'mug', 'pens'] because that is in alphabetical order. 

  2. items = ['fan', 'ball', 'book']
weights = [1, 2, 3]
n = 2

Returns ['ball fan', 'book']
    # When we order the items by weight we get ['book', 'ball', 'fan'], then we put them in our boxes to get the list ['book', 'ball fan'], finally we return ['ball fan', 'book'] because that is in alphabetical order. 

  3. items = ['barbie', 'bear', 'car', 'ball']
weights = [1, 1, 1, 1]
n = 2

Returns ['ball bear', 'barbie car']
    # When we order the items by weight we get ['ball', 'barbie', 'bear', 'car'], then we put them in our boxes to get the list ['ball bear', 'barbie car'], finally we return ['ball bear', 'barbie car'] because that is in alphabetical order.

  4. items = ['picture', 'fan', 'bear', 'candle', 'doll']
weights = [3, 2, 4, 4, 3]
n = 4

Returns ['bear', 'candle', 'doll fan', 'picture']
    # When we order the items by weight we get ['bear', 'candle', 'doll', 'picture', 'fan'], then we put them in our boxes to get the list ['bear', 'candle', 'doll fan', 'picture'], finally we return ['bear', 'candle', 'doll fan', 'picture'] because that is in alphabetical order.
